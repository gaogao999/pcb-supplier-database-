# 基板メーカーDB (Substrate Maker Database)

訪問した基板（PCB）メーカーの技術情報を **少しずつ蓄積・更新していく「生きたデータベース」** のWebアプリです。
`KAGA基板工場調査表 R2` の全項目（19カテゴリ・155項目）をそのままスキーマ化しています。

アイコン付きの実アプリ風UI。インストール不要、ブラウザで `index.html` を開くだけで動きます。

## できること

- 🏭 **メーカー一覧** — カード表示、評価（★）、タグ、入力率、更新日
- 🔎 **検索・絞り込み** — 名前 / 所在地 / タグ、並び替え（更新日・名前・評価・入力率）
- 📋 **詳細ビュー** — 19カテゴリをアコーディオンで表示、記入済み項目数と進捗バー
- 📝 **編集フォーム** — 数値・選択は「ラベル＋入力」を1行で。備考は項目ごとではなく**工程（大項目）ごとに1つ**にまとめてスッキリ
- 🔢 **数値は単位付き数値入力/選択** — 板厚・銅箔厚みは選択式、ドリル径/ランド/公差/L・S・銅厚/BGAピッチ/台数/分解能など約50項目は数値キーボード＋単位表示（φ・±も自動付与）で入力。L/SはL・S・銅厚の3分割、サイズはX・Yの2分割
- 🧱 **工程ごとの構造化行入力** — デスミア（ウェット/ドライ別）、無電解銅メッキ（薬液メーカー・プロセス種類・構成・ライン数・規格）、電解銅メッキ（メーカー・構成・厚み精度・Min製品厚み）、内層/外層前処理（粗化方式・台数）を、項目を分けた行入力で登録・行追加
- ⚡ **ワンタップ選択チップ** — 可/不可・有り/無し・工法・種類などの選択項目はタップだけで回答（選択式は回答の再入力欄を廃止し、チップ＋備考のみ・1行横スクロールで見やすく）
- ☑️ **複数選択対応** — 梱包方法・パターン工法・塗工方法などは複数タップで選択可。可/不可・有無は単一選択
- 🏭 **設備メーカーの行入力** — NCドリル/レーザー/LDI/DI/AVIの装置欄は「メーカー・型番・運転方式(Auto/Semi/Manu)・台数」を横一列＋備考で入力。行追加で複数装置に対応
- 🧪 **材料・薬液メーカーの行入力** — 基材/ドリルビット/SR/シルク材料（材料）、無電解/電解銅メッキ（薬液）を「メーカー・品番・備考」で入力、行追加可
- 💡 **メーカー候補サジェスト（工程・種別ごと）** — メーカー欄にフォーカスすると、その工程の候補だけを入力欄の直下に横並びチップで表示。候補は⚙️設定ページで**工程ごと・種別（設備/材料/薬液）ごと**に登録でき、過去入力からも自動収集
- 👆 **行はスワイプ操作** — 装置/材料/薬液などの行は、左スワイプで「削除」、右スワイプで「複製」（iOSメール風）。工程切替スワイプとは競合しない
- 🧭 **基本情報と工程入力を分離** — 追加時は基本情報（会社名など）だけの軽いフォーム。保存すると自動で工程1の入力シートが開き、そのままスワイプで全工程を連続入力できる
- 👈 **スワイプで工程切替** — 詳細画面・編集シートとも左右スワイプ＋◀▶ボタン（n/19表示付き）で前後の工程へ（編集中は自動保存）。PCは←→キー
- 👍 **親指ゾーンの編集ボタン** — 各工程カードの下部に「この工程を編集」ボタン
- 💾 **下書き自動保存** — 入力途中で閉じても次回開いたときに復元できる
- 🛡 **データ保護** — 保存のたびに直前状態を、1日1回その日の開始状態を自動バックアップ。破損時は自動復旧。アプリを更新してもデータは消えません
- 🔢 **バージョン表示** — ヘッダーと一覧下部に表示（現在 v2.0.0）
- 👁 **未記入を隠す** — 詳細画面で記入済みの項目だけを表示に切替
- 📍 **訪問記録（タイムライン）** — 訪問日・記録者・所感を追記。「生きた情報」を時系列で残せる
- ⚖️ **比較** — 最大3社を横並びで比較（両社とも空欄の項目は自動で非表示）
- 🎨 **紙面×ウルトラマリンのUI** — 温かい紙面(#F4F1EA)×藍アクセント、日本語=Zen Kaku Gothic New / 数値=JetBrains Mono（等幅で桁が揃う）
- 🌗 **ライト / ダークテーマ**
- 💾 **JSON 書き出し / 読み込み** — データを持ち出し、repo に残す・共有する・端末間で移す

## 使い方

1. `index.html` をブラウザで開く（GitHub Pages で公開も可）
2. 右上「＋追加」で訪問したメーカーを登録
3. 訪問のたびに「編集」で項目を更新し、「訪問記録を追加」で所感を残す
4. 定期的に **⬇ JSON書き出し** でバックアップ。`data/makers.json` として repo にコミットすれば履歴も残せます

## データの保存場所

- 入力データはブラウザの **localStorage** に保存されます（端末内）。
- 初回起動時は `data/makers.sample.json`（調査表の記入例 = TSB工場）を読み込みます。
- 端末をまたぐ・チームで共有する場合は **JSON書き出し → 読み込み** を使ってください。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` | アプリ本体 |
| `styles.css` | スタイル（ライト/ダーク対応） |
| `app.js` | ロジック（一覧・詳細・編集・訪問記録・比較・入出力） |
| `schema.js` | 調査項目スキーマ（調査表から自動生成） |
| `icon.svg` | アプリアイコン（基板モチーフ） |
| `manifest.webmanifest` | PWA用（ホーム画面に追加可） |
| `data/makers.sample.json` | サンプルデータ（記入例） |
